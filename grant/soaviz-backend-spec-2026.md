> **"SOAVIZ는 브라우저 데모가 아니라, 어느 기기에서 로그인해도 창작 세계가 그대로 있는 서비스다."**

### As-Is 진단 (정직한 현황)

| 레이어 | 상태 | 2R 리스크 |
|---|---|---|
| 프론트엔드 (soaviz.com/app · /team-os) | ✅ 완성 — 워크플로우·크레딧 UI·entitlement·대시보드 | 없음 |
| 데이터 영속 | ⚠ **전부 localStorage** — 기기 종속, 위변조 가능, 유실 위험 | **치명** — "Context 보존" 주장과 모순 |
| 인증 | ⚠ Google OAuth 절반 (콜백 미설정, 세션-데이터 미연결) | 높음 — 심사위원이 가입해보면 들통 |
| 크레딧/한도 | ⚠ 클라이언트 전용 (서버 검증 0) | 중 — 새로고침이면 리셋되는 숫자 |
| DB 스키마 | ⚠ SQL 작성됨, **미적용** (`supabase/migration-2026-06-10-credits.sql`) | — |
| 문의 접수 | ⚠ mailto 의존 | 중 — 유실 가능 |

**결론**: 프론트에 1원도 쓰지 않는다. 200만원 전액 = 서버 레이어. 심사 데모의 킬샷은 **"노트북에서 작업 → 폰에서 로그인 → 같은 자리에서 이어가기"** 30초 시연이다.

## 1. To-Be 아키텍처

```
[브라우저: soaviz.com/app · /team-os · /mobile]
   │  Supabase JS SDK (이미 로드됨)
   ▼
[Supabase (Seoul) — 단일 백엔드]
   ├─ Auth: Google OAuth → 세션 → 지갑·프로필 자동 생성 (DB trigger)
   ├─ Postgres + RLS (행 단위 본인 데이터 격리)
   │    profiles / projects / characters / prompts / assets
   │    versions / credit_wallet / credit_transactions
   │    plan_entitlements / inquiries
   ├─ RPC: spend_credits() · reset_monthly_credits() (작성 완료, 배선 대상)
   └─ Storage: 버킷 soaviz-assets (project-id/{images,videos,audio,exports})
   
[동기화 엔진 — 이번 용역의 심장]
   localStorage(기존 키 그대로) ⇄ Supabase
   · 로그인 시 풀(pull) → 병합 → 주기적/변경시 푸시(push, 2s 디바운스)
   · 충돌: updated_at 기준 last-write-wins + 변경 로그 보존
   · 오프라인: 로컬 우선, 재접속 시 큐 플러시
```

설계 원칙 3개: ① 기존 단일 HTML·localStorage 키 구조를 **바꾸지 않고** 그 아래에 동기 레이어를 깐다(프론트 재작업 0) ② 서버가 진실의 원천(한도·크레딧), 로컬은 캐시 ③ 모든 테이블 RLS — Team OS 보안 서사와 일관.

## 3. 모듈별 검수 기준 (Acceptance Criteria)

외주 검수 시 아래 항목이 전부 ✓여야 잔금 지급. 항목 자체가 정산 "수행결과 증빙"이 된다.

**M1**
- [ ] Supabase 대시보드에서 11개 테이블 + RLS 정책 확인 가능
- [ ] 타 계정 토큰으로 내 행 조회 시 0건 (RLS 차단 스크린샷)

**M2**
- [ ] soaviz.com에서 구글 로그인 → 팝업 자동 닫힘 → 앱 세션 유지
- [ ] 신규 가입 직후 credit_wallet에 free/100cr 행 자동 생성
- [ ] 모바일(/mobile)에서 동일 세션 인식

**M3** ★
- [ ] 기기 A에서 프로젝트·캐릭터·프롬프트 생성 → 기기 B 로그인 시 60초 내 동일 표시
- [ ] 오프라인에서 수정 → 재접속 시 자동 반영
- [ ] 동시 수정 충돌 시 최신본 유지 + 이전본이 versions에 보존
- [ ] 기존 비로그인 사용 데이터가 로그인 시 서버로 1회 이관(마이그레이션)

**M4**
- [ ] 영상 생성 실행 시 credit_transactions에 차감 행 기록
- [ ] 잔액 부족 시 서버가 거부(클라 변조로 우회 불가) + 안내 모달
- [ ] 실패한 생성은 차감 0 (보장 문구와 일치)

**M5**
- [ ] 업로드 자산이 project-id 경로에 저장, 메타에 출처 프롬프트·캐릭터·모델·시드 연결
- [ ] 자산 상세에서 "이 결과를 만든 프롬프트" 역추적 가능

**M6**
- [ ] Team OS 문의 폼 제출 → inquiries 행 생성 → 관리자 화면에서 목록 확인

**M7**
- [ ] 2기기 시연 시나리오 영상 1편(30초~1분) + 문서 4종

## 6. 2라운드 데모 스크립트 연결 (이 용역의 최종 목적)
