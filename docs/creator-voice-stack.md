# 크리에이터 보이스 앱 기술 스택 제안

한국 크리에이터 대상 음성 추출, 자막, 보이스 클로닝, 본인 인증, 워터마킹까지 확장하는 것을 기준으로 정리한 권장 스택입니다.

## 추천 스택

| 레이어 | 추천 (Best) | 대안 | 선정 이유 |
|---|---|---|---|
| 보이스 클로닝 (MVP) | ElevenLabs IVC/PVC | Resemble AI / PlayHT | 한국어 품질이 안정적이고, 워터마킹 및 모니터링 기능이 비교적 잘 갖춰져 있음 |
| 보이스 클로닝 (확장) | F5-TTS | XTTS v2 / OpenVoice V2 | 2024 기준 성능이 강하고 한국어 튜닝 여지가 크며 Modal GPU 비용 구조에도 잘 맞음 |
| 음성 품질 검증 | pyannote.audio + SNR 측정 | Resemblyzer | 노이즈 상태, 화자 일관성, 세그먼트 품질을 자동 검수하기 좋음 |
| 본인 인증 (Liveness) | 랜덤 문장 낭독 비교 (Resemblyzer) | 화자 임베딩 비교 | 업로드한 목소리와 실시간 녹음 화자가 같은지 확인하는 흐름에 적합 |
| 워터마킹 | AudioSeal (Meta) | Silent watermark | 생성물 추적과 오남용 대응 측면에서 유리 |
| 텍스트 에디터 | TipTap + SSML 어댑터 | Slate | 강조, 일시정지, 감정 태그를 UI로 자연스럽게 붙이기 쉬움 |

## 권장 조합

### 1. 빠른 MVP

- 음성 추출: `yt-dlp` + `ffmpeg` + `demucs`
- 자막 생성: Whisper 또는 `faster-whisper`
- 보이스 클로닝: ElevenLabs IVC/PVC
- 본인 확인: 랜덤 문장 읽기 + Resemblyzer 비교
- 결과 추적: AudioSeal

이 조합은 구현 속도와 한국어 품질, 운영 리스크 관리의 균형이 좋습니다.

### 2. 자체 호스팅 확장형

- 음성 분리: Modal + `demucs`
- 전사: Modal + `faster-whisper`
- 보이스 생성: F5-TTS
- 화자 분리 및 품질 검수: `pyannote.audio`
- 워터마킹: AudioSeal

이 조합은 장기적으로 비용 최적화와 한국어 특화 튜닝에 유리합니다.

## 추천 아키텍처 방향

### 크리에이터 기본 흐름

1. URL 또는 파일 입력
2. 음성 추출
3. Whisper 자막 생성
4. 필요 시 화자 분리
5. 결과 다운로드
6. 선택적으로 보이스 클로닝 또는 TTS 편집으로 이동

### 보이스 클로닝 안전 흐름

1. 사용자 음성 업로드
2. 랜덤 문장 낭독 녹음
3. 화자 일치 검증
4. 통과 시 보이스 생성 허용
5. 생성 결과에 워터마킹 삽입
6. 로그와 메타데이터 저장

## 우선순위 제안

### Phase 1

- 음성 추출
- Whisper 자막 생성
- 24시간 삭제 정책
- 저작권 고지

### Phase 2

- 화자 분리
- 본인 인증
- 워터마킹

### Phase 3

- 보이스 클로닝
- 감정 태그 에디터
- 크레딧 및 결제 시스템

## 구현 메모

- MVP 단계에서는 외부 API 기반 품질 확보가 중요하므로 ElevenLabs가 유리합니다.
- 장기적으로는 Modal 위에 F5-TTS, `faster-whisper`, `demucs`를 올려서 GPU 워크로드를 통합하는 방향이 비용상 더 좋습니다.
- `pyannote.audio`는 화자 분리뿐 아니라 품질 검수 파이프라인에도 재사용 가치가 큽니다.
- AudioSeal 같은 워터마킹은 출시 초기에 바로 넣어두는 편이 정책 대응에 도움이 됩니다.
