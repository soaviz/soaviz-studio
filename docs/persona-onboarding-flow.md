# 페르소나 온보딩 플로우

추출 결과 이후 사용자가 `페르소나 만들기`를 눌렀을 때 이어지는 3단계 온보딩 플로우 문서입니다.

대상 단계:

- ④ 보이스 검수/인증
- ⑤ 페르소나 등록
- ⑥ TTS 스튜디오

## 한눈에 보는 흐름

```text
③ 추출 결과
   -> 페르소나 만들기 CTA
   -> ④ 보이스 검수/인증
   -> ⑤ 페르소나 등록
   -> ⑥ TTS 스튜디오
```

## 제품 목표

- 사용자가 “내 목소리로 안전하게 페르소나를 만들 수 있다”는 신뢰를 느끼게 한다
- 검수 단계를 위협적으로 보이지 않게, 자연스럽고 짧은 절차로 안내한다
- 인증 직후 바로 페르소나를 완성하고 실제 TTS 생성까지 이어지게 한다

## 전체 온보딩 구조

### Step 4. 보이스 검수/인증

목적:

- 업로드 음성의 품질을 빠르게 확인
- 업로드한 목소리와 실시간 녹음 화자가 같은지 확인

핵심 요소:

- 파형
- SNR 점수
- 품질 상태
- 랜덤 문장 낭독
- 화자 일치 결과

완성형 카피:

| 요소 | 카피 |
|---|---|
| H1 | 내 목소리, 맞는지 확인할게요 |
| Sub | 클론 생성 전에 2단계 검수를 거칩니다 |
| 품질 섹션 제목 | 음성 품질 점수 |
| 품질 낮음 알림 | 노이즈가 많아요. 더 조용한 구간을 골라주세요. |
| 본인 인증 제목 | 본인 인증 |
| 지시 | 아래 문장을 15초 안에 녹음해 주세요 |
| 랜덤 문장 설명 | 이 문장은 매번 무작위로 바뀌어요. 녹음은 서버에 저장되지 않아요. |
| 성공 | 동일 화자로 확인됐어요 (일치도 94%) |
| 실패 | 다른 사람의 목소리로 보여요. 다시 녹음해 주세요. |

와이어프레임:

```text
┌─────────────────────────────────────────┐
│  ← 뒤로                      [건너뛰기] │
├─────────────────────────────────────────┤
│                                         │
│       내 목소리, 맞는지 확인할게요       │
│   클론 생성 전에 2단계 검수를 거칩니다.  │
│                                         │
│   ─────────────────────────────────     │
│   1. 품질 점수                           │
│      음질     ●●●●○  Good               │
│      선명도   ●●●●●  Excellent          │
│      길이     1분 24초 (권장 30초↑)      │
│   ─────────────────────────────────     │
│   2. 본인 인증                           │
│                                         │
│   아래 문장을 15초 안에 녹음해 주세요.   │
│   ┌─────────────────────────────────┐   │
│   │ "사월의 바람은 창문을 지나        │   │
│   │  낯선 이름을 부른다."            │   │
│   └─────────────────────────────────┘   │
│                                         │
│            [ ● 녹음 시작 ]              │
│                                         │
│   이 문장은 매번 무작위로 바뀌어요.     │
│                                         │
├─────────────────────────────────────────┤
│   [ 다음: 페르소나 만들기 ]              │
└─────────────────────────────────────────┘
```

### Step 5. 페르소나 등록

목적:

- 인증이 끝난 목소리에 이름과 정체성을 부여
- 사용자가 이 목소리를 “내 캐릭터/내 브랜드”로 받아들이게 함

핵심 요소:

- 이름
- 설명
- 샘플 프리뷰 3종
- 저장 CTA

추천 구성:

- 이름 입력
- 한 줄 설명
- 톤 요약 태그
- 샘플 프리뷰 3종
  - 뉴스
  - 내레이션
  - 대화
- `페르소나 등록하기` 버튼

완성형 카피:

| 요소 | 카피 |
|---|---|
| H1 | 이 목소리를 어떻게 소개할까요? |
| Sub | 이름과 설명을 붙이면 나만의 페르소나로 저장할 수 있어요. |
| 이름 레이블 | 페르소나 이름 |
| 이름 플레이스홀더 | 예: 은교 내레이션 |
| 설명 레이블 | 한 줄 설명 |
| 설명 플레이스홀더 | 예: 차분하고 또렷한 브랜드 내레이션 톤 |
| 샘플 섹션 제목 | 샘플로 먼저 들어보세요 |
| 샘플 1 | 뉴스 |
| 샘플 2 | 내레이션 |
| 샘플 3 | 대화 |
| Primary CTA | 페르소나 등록하기 |

와이어프레임:

```text
┌─────────────────────────────────────────┐
│  ← 뒤로                                 │
├─────────────────────────────────────────┤
│                                         │
│      이 목소리를 어떻게 소개할까요?      │
│ 이름과 설명을 붙이면 나만의 페르소나로   │
│ 저장할 수 있어요.                        │
│                                         │
│  페르소나 이름                           │
│  [ 은교 내레이션                    ]    │
│                                         │
│  한 줄 설명                              │
│  [ 차분하고 또렷한 브랜드 내레이션 톤 ]  │
│                                         │
│  샘플로 먼저 들어보세요                  │
│  [ 뉴스 재생 ] [ 내레이션 재생 ] [ 대화 ]│
│                                         │
├─────────────────────────────────────────┤
│      [ 페르소나 등록하기 ]               │
└─────────────────────────────────────────┘
```

### Step 6. TTS 스튜디오

목적:

- 등록된 페르소나로 바로 첫 생성 경험을 제공
- 기능보다 “내 목소리로 진짜 만들어졌다”는 감각을 먼저 전달

핵심 요소:

- 텍스트 에디터
- 감정 슬라이더
- 속도/피치 조절
- 생성 버튼
- 히스토리

추천 구성:

- 페르소나 헤더
- 텍스트 입력창
- 감정 슬라이더
- 속도 조절
- 피치 조절
- 생성 버튼
- 생성 결과 리스트

완성형 카피:

| 요소 | 카피 |
|---|---|
| H1 | 내 페르소나로 바로 만들어볼게요 |
| Sub | 텍스트를 입력하면 내 목소리 톤으로 자연스럽게 읽어줘요. |
| 입력 플레이스홀더 | 읽을 문장을 입력해 주세요 |
| 감정 레이블 | 감정 |
| 속도 레이블 | 속도 |
| 피치 레이블 | 피치 |
| Primary CTA | 음성 생성하기 |
| 히스토리 제목 | 최근 생성한 음성 |
| 빈 상태 | 아직 생성한 음성이 없어요 |

와이어프레임:

```text
┌─────────────────────────────────────────┐
│  은교 내레이션                          │
├─────────────────────────────────────────┤
│                                         │
│     내 페르소나로 바로 만들어볼게요      │
│ 텍스트를 입력하면 내 목소리 톤으로       │
│ 자연스럽게 읽어줘요.                     │
│                                         │
│  [ 읽을 문장을 입력해 주세요          ]  │
│                                         │
│  감정   [────●────]                     │
│  속도   [───●─────]                     │
│  피치   [─────●───]                     │
│                                         │
│        [ 음성 생성하기 ]                │
│                                         │
│  최근 생성한 음성                        │
│  아직 생성한 음성이 없어요               │
│                                         │
└─────────────────────────────────────────┘
```

## 세 단계를 하나로 보는 UX 원칙

- 검수는 신뢰 확보 단계다
- 등록은 정체성 부여 단계다
- TTS는 즉시 보상 단계다

즉, 감정 흐름은 이렇게 가져가는 것이 좋습니다.

1. `이 목소리가 정말 내 것인지 확인`
2. `이 목소리에 이름과 성격을 부여`
3. `그 목소리로 바로 결과를 만들어봄`

## 화면 간 전환 규칙

- ④ 통과 시 자동으로 ⑤로 이동
- ④ 실패 시 같은 화면에서 재녹음
- ⑤ 완료 시 성공 토스트와 함께 ⑥ 진입
- ⑥에서는 첫 생성 예시 문장을 기본값으로 미리 넣어둘 수 있음

## 권장 기본 예시 문장

### 뉴스

`오늘 주요 소식을 차분하고 또렷하게 전해드리겠습니다.`

### 내레이션

`사월의 바람은 창문을 지나 낯선 이름을 부른다.`

### 대화

`안녕하세요, 오늘은 새로 만든 페르소나 목소리를 함께 들어볼게요.`

## 구현 우선순위

### 1차

- ④ 보이스 검수 화면
- ⑤ 페르소나 등록 화면
- ⑥ TTS 스튜디오 진입 연결

### 2차

- 샘플 프리뷰 3종
- 감정 슬라이더
- 최근 히스토리

### 3차

- 고급 감정 태그
- SSML 어댑터
- 워터마킹/정책 레이어
