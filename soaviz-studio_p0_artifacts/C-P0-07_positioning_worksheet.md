# C-P0-07: 제품 포지셔닝 결정 워크시트

## 결정 사항

SOAVIZ STUDIO의 메인 진입 흐름을 다음 3안 중 하나로 확정.

## 3안 비교

### A안: Pro 단독 (★★★★★ 추천)

**구조**
- landing의 "3가지 편집안" Quick Mode 제거
- 모든 신규 사용자 → projects/personas로 직접 진입
- Pipeline 5단계 워크플로우만 노출

**장점**
- 일관된 UX
- ORA-MU/천상유람 도그푸딩 시나리오와 정합
- 코드 분기 최소화

**단점**
- 진입 장벽 높음 (학습 곡선)
- 첫 30초 전환율 저하 가능

**작업 시간**: 2시간 (landing 단순화)

---

### B안: Quick Mode 흡수 (★★★)

**구조**
- landing의 "3가지 편집안" UI를 Pipeline 안의 "Quick Mode 토글"로 이동
- 기본은 Pro Mode, 클릭 시 Quick으로 전환

**장점**
- 두 사용자층 모두 지원
- 저장된 프로젝트 + 빠른 시연 모두 가능

**단점**
- UI 모드 토글 복잡도 ↑
- Quick Mode가 작동하려면 별도 구현 필요

**작업 시간**: 1주 (Quick Mode 별도 구현)

---

### C안: 두 진입 분리 (★★)

**구조**
- /quick 라우트 (3가지 편집안)
- /pro 라우트 (Pipeline)
- landing에서 둘 중 선택

**장점**
- 타깃별 최적화

**단점**
- 코드 분기 폭증
- 마케팅/SEO/온보딩 모두 두 벌

**작업 시간**: 2-3주

---
